## Technical Debt (Non-Blockers)

### Phase 11: Build & Packaging

| Issue | Impact | Mitigation |
|-------|--------|------------|
| pkg tool does not support ESM modules with import.meta | Standalone .exe binaries not viable | Distribution via `npm install -g .` |
| pkg only supports Node.js v12-v16 | Project requires v20+ | ESM build works with modern Node |
| Native module bundling fails (conf uses node:sqlite) | pkg cannot bundle conf package | Users need Node.js runtime |

**Resolution:** Documented that ESM-first distribution is the primary strategy. pkg binaries are documented as experimental.

### Phase 09: GitHub Actions Integration

| Issue | Impact | Mitigation |
|-------|--------|------------|
| Renderer repo (autocliper-renderer) is separate | Workflow file not in this codebase | Manual setup documented in README |

**Resolution:** This is by design - CLI and renderer are separate repositories.

### Phase 07: Face Detection

| Issue | Impact | Mitigation |
|-------|--------|------------|
| Python + MediaPipe are optional | Not auto-tested in CI | Graceful fallback to CENTER mode |

**Resolution:** Face detection is optional enhancement, not required for core functionality.

## Known Limitations

1. **Standalone Binaries:** pkg tool cannot create working standalone executables due to ESM and native module limitations. Users must have Node.js 20+ installed.

2. **Python Dependency:** Face detection requires Python and MediaPipe installation. The CLI gracefully falls back to CENTER mode if unavailable.

3. **Renderer Setup:** The autocliper-renderer repository must be set up separately with its own GitHub Actions workflow file.

4. **API Costs:** Users must provide their own API keys for Deepgram, Gemini, and a GitHub token with Actions workflow permissions.
